Основные команды и действия

Начало работы

Устанавливаем gulp: npm install gulp

Проверяем текущие установленные версии:

node --version (-v)

npm -v

npx -v

После создания папки под проект, создаем файл .json

npm init (-y)

Устанавливаем пакет gulp в наш проект

npm install --save-dev gulp

Проверяем текущую версию gulp: gulp --version

Создаем файл gulpfile.js

!Важно. "devDependencies" и "dependencies"

"devDependencies" - для плагинов и разработки0 то что не попадет в финальную версию (префиксы, минификация и тд)

"dependencies" - подключение скриптов (bootstrap, typeScript, Jquary и тд), то что попадет в финальную версию проекта

Устанавливаем плагин less

npm i less -D

Устанавливаем плагин del

npm i del -D

В gulpfile.js подключаем gulp и less

const gulp = require('gulp');

const less = require('less');

const del = require('del');

Создаем первую функцию для удаления файлов в dist (финальная папка проекта)

    function clean() {
      return del(['dist'])
    }

Создаем корневую папку для разработки проекта src

Создаем корневую папку под финальную сборку проекта dist

Создаем объект path с путями к файлам

Также создаем каталоги с файлами соотвественно (main.min)

Создаем функцию styles

Создаем функцию watch

Ctr + C -- выход из вотчера

Метода .series и .parallel

Используем .series для последовательного выполнения задач (тасков)

Используем .parallel для параллельного выполнения задач (css и js)

Создаем build

Добавляем в экспортс default -- gulp запускает build
